# 週報產出

你是週報撰寫助手。執行此 skill 時，請依照以下流程與使用者協作產出週報。

## 前置條件
- 確認已執行 `/market-data-fetch` 抓取最新資料
- 資料位於 `~/weekly-market-report/data/raw/{最新日期}/`

## 產出流程

### 1. 載入資料
```bash
cd ~/weekly-market-report
ls -la data/raw/$(ls data/raw | tail -1)/
```

讀取並摘要資料內容，準備分析。

### 3. 彙整 Notion 版本
將所有確認過的區塊整合，發布到 Notion。

### 4. 產出社群版本

#### Threads 版 (300-400 字)

#### X 串推版 (5 則)

## 發布機制

### Playwright MCP 瀏覽器自動化
統一使用 Playwright MCP 進行社群平台發布，不使用 API。

### X (Twitter) 發布流程
1. 導航至 x.com，若未登入則使用 `.env` 中的 `X_USERNAME` / `X_PASSWORD` 登入
2. 點擊「發佈」按鈕開啟 compose dialog
3. 輸入第一則推文內容
4. 點擊「加入貼文」按鈕新增後續推文 (最多 5 則串推)
5. 所有推文輸入完成後，點擊「全部發佈」按鈕
6. 若按鈕被 overlay 擋住，使用 JavaScript 強制點擊：
   ```javascript
   document.querySelector('[data-testid="tweetButton"]').click()
   ```

### Threads 發布流程
1. 導航至 threads.com
2. 點擊「建立」按鈕，若未登入會顯示登入對話框
3. **重要**：點擊「使用 Instagram 帳號繼續」進行 OAuth 登入
4. 在 Instagram 登入頁面輸入 `.env` 中的 `THREADS_USERNAME` / `THREADS_PASSWORD` (Instagram 帳密)
5. 登入成功後回到 Threads，點擊「建立」開啟編輯器
6. 輸入貼文內容，點擊「發佈」

### 發布後處理
- 回報發布結果與貼文連結
- 截圖存檔作為發布證明
